Migration Guide · v0.11 → v0.12

Migrating to SDK v0.12

Three breaking changes. Most callers need to update transport config structs, update type assertions on Read/Await/Subscribe results, and replace direct field access on *Client.

Breaking changes

1. Typed transport configs replace string-keyed maps

v0.11 — transport was passed as TransportType and TransportDir fields or as a map[string]any.

v0.12 — pass a typed config struct implementing the Transport interface.

// Before (v0.11)
client.Create(protocol.CreateRequest{
    TransportType: "filesystem",
    TransportDir:  "/path/to/campfires",
})

// After (v0.12)
client.Create(protocol.CreateRequest{
    Transport: protocol.FilesystemTransport{Dir: "/path/to/campfires"},
})

The same change applies to JoinRequest, AdmitRequest, and EvictRequest. The three available transport types:

// Filesystem
protocol.FilesystemTransport{Dir: "/path/to/campfires"}

// P2P HTTP
protocol.P2PHTTPTransport{
    Transport:    httpTransport,         // *cfhttp.Transport, already started
    MyEndpoint:   "http://host:9001",
    PeerEndpoint: "http://peer:9001",    // required for Join
    Dir:          "/optional/state/dir",
}

// GitHub
protocol.GitHubTransport{
    Owner: "org", Repo: "repo", Branch: "main",
    Dir: "campfires/", Token: os.Getenv("GITHUB_TOKEN"),
}

2. Read, Await, and Subscribe return protocol.Message

v0.11 — Read returned []store.MessageRecord, Await returned *store.MessageRecord, and Subscribe emitted store.MessageRecord values.

v0.12 — all three return protocol.Message (or []protocol.Message). This is an SDK-stable type that adds IsBridged() and removes the dependency on pkg/store from caller code.

// Before (v0.11) — result.Messages was []store.MessageRecord
for _, r := range result.Messages {
    fmt.Println(r.Sender, string(r.Payload))
}

// After (v0.12) — result.Messages is []protocol.Message
for _, m := range result.Messages {
    fmt.Println(m.Sender, string(m.Payload))
    if m.IsBridged() {
        // message came through a blind-relay hop
    }
}

Await similarly returns *protocol.Message. Subscribe channels now emit protocol.Message values. Field names are unchanged; the only difference is the type package (protocol vs store).

3. Store and Identity fields removed from *Client

v0.11 — client.Store and client.Identity were exported fields, allowing direct access to the underlying store and keypair.

v0.12 — both are unexported. Use the provided accessors:

// Before (v0.11)
pubKey := client.Identity.PublicKeyHex()

// After (v0.12)
pubKey := client.PublicKeyHex() // returns "" for read-only clients

If you need the store for low-level access, use client.ClientStore() (returns store.Store). Prefer Client methods for all standard operations.

New features in v0.12

protocol.Message and IsBridged()

The new SDK-facing message type with IsBridged() — returns true when the message passed through a blind-relay provenance hop, indicating it was bridged from an external system (Teams, Slack, etc.).

client.Get and client.GetByPrefix

msg, err := client.Get(messageID)         // nil, nil if not found
msg, err := client.GetByPrefix("a1b2c3") // error if ambiguous prefix

client.PublicKeyHex()

pubKey := client.PublicKeyHex() // pass to convention.NewExecutor, etc.

CreateRequest.Description and CreateResult.BeaconID

result, err := client.Create(protocol.CreateRequest{
    Transport:   protocol.FilesystemTransport{Dir: dir},
    Description: "my coordination campfire",
})
fmt.Println(result.BeaconID) // hex beacon ID — equals CampfireID

Dual-loop pattern

Running convention.Server and Subscribe concurrently in the same process using two separate Client instances. See the SDK reference for the full example.

Quick checklist

  • Replace TransportType/TransportDir map fields with typed transport structs in all CreateRequest, JoinRequest, AdmitRequest, and EvictRequest calls.
  • Update Read/Await/Subscribe callers: change type assertions from store.MessageRecord to protocol.Message.
  • Replace client.Identity.PublicKeyHex() with client.PublicKeyHex().
  • Remove any direct client.Store or client.Identity field access.
  • Pass client.PublicKeyHex() to convention.NewExecutor.
