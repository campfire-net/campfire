Campfire Go SDK

Build services on campfire. Start with an LLM handler, move to CPU code — transparently to callers.

pkg/protocol provides a Client for the full campfire lifecycle. pkg/convention layers typed operation dispatch on top. Both packages work across all transports without transport-specific code in your service.

New in v0.12: Typed transport configs, protocol.Message type with IsBridged(), Get/GetByPrefix methods, PublicKeyHex(), CreateRequest.Description, and the dual-loop pattern for concurrent Subscribe + convention.Server. See the migration guide if upgrading from 0.11.

Init

One call to get started. Generates or loads an Ed25519 identity, opens a SQLite store, returns a ready *Client.

client, err := protocol.Init("~/.campfire")
// Pass "" for the default path (~/.campfire).
defer client.Close()

Init is idempotent — calling it twice with the same path returns a client with the same identity.

For explicit control (custom store backend):

id, _ := identity.Load("/path/to/identity.json")
s, _  := store.Open("/path/to/store.db")
client := protocol.New(s, id)

Client

Client is the primary API surface. All operations — create, join, send, read, subscribe, member management — go through it regardless of transport.

Not safe for concurrent use. Use one Client per goroutine. For concurrent loops (e.g., Subscribe + convention.Server), create two separate clients from the same configDir — see the dual-loop pattern.

pubKey := client.PublicKeyHex() // hex-encoded Ed25519 public key; "" for read-only clients

Full method surface:

protocol.Init(cfHome) → *Client
client.Create(CreateRequest) → CreateResult
client.Join(JoinRequest) → JoinResult
client.Leave(campfireID)
client.Disband(campfireID)
client.Admit(AdmitRequest)
client.Evict(EvictRequest) → EvictResult
client.Members(campfireID) → []MemberRecord
client.Send(SendRequest) → *Message
client.Read(ReadRequest) → *ReadResult
client.Get(id) → *Message
client.GetByPrefix(prefix) → *Message
client.Await(AwaitRequest) → *MessageRecord
client.Subscribe(ctx, SubscribeRequest) → *Subscription
client.PublicKeyHex() → string
client.Close() error

Message type

protocol.Message is the SDK-facing message type returned by Read, Get, GetByPrefix, Await, and Subscribe. New in v0.12.

type Message struct {
    ID          string
    CampfireID  string
    Sender      string   // hex-encoded Ed25519 public key [verified]
    Payload     []byte   // [TAINTED] sender-controlled content
    Tags        []string // [TAINTED] sender-asserted labels
    Antecedents []string // [TAINTED] causal claims, not proofs
    Timestamp   int64    // [TAINTED] sender wall clock
    Instance    string   // [TAINTED] sender-asserted role label
    Signature   []byte   // [verified]
    Provenance  []message.ProvenanceHop // [verified]
}

Tainted fields (Payload, Tags, Antecedents, Instance, Timestamp) are sender-asserted and must not be the sole basis for trust decisions or access control. Sender, Signature, and Provenance are cryptographically verified.

IsBridged()

if msg.IsBridged() {
    // at least one provenance hop is a blind-relay hop
    // message originated from an external system (Teams, Slack, etc.)
}

IsBridged() returns true when at least one provenance hop carries the blind-relay role. Use it to identify messages that arrived through a bridge rather than from a native campfire participant.

MessageFromRecord

// Convert a store.MessageRecord to protocol.Message (for use with IsBridged, etc.)
msg := protocol.MessageFromRecord(record)

Transports

The Transport interface is sealed — only three concrete types are accepted. Pass one to CreateRequest.Transport or JoinRequest.Transport. After creation/join, transport selection is automatic — the campfire's membership record determines routing.

type Transport interface {
    TransportType() string // "filesystem" | "p2p-http" | "github"
}

FilesystemTransport

protocol.FilesystemTransport{
    Dir: "/path/to/campfires", // base directory for campfire state
}

Members share a directory. Best for single-machine multi-agent setups, CI, and local development.

P2PHTTPTransport

protocol.P2PHTTPTransport{
    Transport:    httpTransport,        // *cfhttp.Transport, started before use
    MyEndpoint:   "http://host:9001",   // this node's public HTTP endpoint
    PeerEndpoint: "http://peer:9001",   // peer endpoint — required for Join, not Create
    Dir:          "/optional/state",    // if empty, a temp directory is used
}

Direct peer-to-peer HTTP delivery. Required for threshold signatures (FROST DKG) with Threshold > 1.

GitHubTransport

protocol.GitHubTransport{
    Owner:  "org",
    Repo:   "repo",
    Branch: "main",
    Dir:    "campfires/",
    Token:  os.Getenv("GITHUB_TOKEN"), // or pass via SendRequest.GitHubToken
}

Messages stored as GitHub issue comments. Useful for campfires that span machines without direct connectivity.

Get and GetByPrefix

Targeted single-message lookups from the local store. New in v0.12.

// Exact ID lookup — returns nil, nil if not found
msg, err := client.Get(messageID)

// Prefix lookup — returns error if the prefix is ambiguous (matches > 1 message)
msg, err := client.GetByPrefix("a1b2c3d4")

Both return *protocol.Message. Use for targeted lookups after you already know the message ID — for example, fetching a specific future by ID after receiving its ID from a peer.

Send

msg, err := client.Send(protocol.SendRequest{
    CampfireID:  campfireID,
    Payload:     []byte("message text"),
    Tags:        []string{"status"},
    Antecedents: []string{priorMsgID}, // optional reply threading
    Instance:    "my-service",         // optional tainted role label
})

Antecedents threads the message into the DAG — readers reconstruct conversation trees from the antecedent graph. Instance is sender-asserted and not covered by the signature; useful for display, not access control.

Role enforcement applies before sending. Observer-role members cannot send. Writer-role members cannot send campfire:* system messages.

var roleErr *protocol.RoleError
if protocol.IsRoleError(err, &roleErr) {
    // membership role prohibits this send
}

Read

result, err := client.Read(protocol.ReadRequest{
    CampfireID:       campfireID,
    Tags:             []string{"status"},       // OR filter — any matching tag
    TagPrefixes:      []string{"galtrader:"},   // OR with Tags — prefix match
    ExcludeTags:      []string{"compaction"},
    Sender:           senderPubKeyHex,          // optional
    AfterTimestamp:   cursor,                   // nanoseconds; 0 = all
    Limit:            50,
    IncludeCompacted: false,
    SkipSync:         false,                    // set true to skip transport sync
})
// result.Messages — []protocol.Message, ordered by timestamp
// result.MaxTimestamp — use as cursor on the next call

Cursor pattern

var cursor int64
for {
    result, _ := client.Read(protocol.ReadRequest{
        CampfireID:     campfireID,
        AfterTimestamp: cursor,
    })
    for _, m := range result.Messages {
        process(m)
    }
    cursor = result.MaxTimestamp
    time.Sleep(5 * time.Second)
}

MaxTimestamp reflects the highest timestamp across all messages (pre-filter), ensuring filtered-out messages don't reappear on subsequent reads. Use Subscribe instead of a manual cursor loop when you need continuous watching.

Await

Blocks until another agent sends a message that fulfills a specific message ID.

fulfillment, err := client.Await(protocol.AwaitRequest{
    CampfireID:   campfireID,
    TargetMsgID:  future.ID,
    Timeout:      30 * time.Second,
    PollInterval: 2 * time.Second, // default 2s
})

if errors.Is(err, protocol.ErrAwaitTimeout) {
    // nobody fulfilled before deadline
}

A fulfilling message must carry the "fulfills" tag and list TargetMsgID in its antecedents:

_, err = peer.Send(protocol.SendRequest{
    CampfireID:  campfireID,
    Payload:     []byte(`{"answer":"done"}`),
    Tags:        []string{"fulfills"},
    Antecedents: []string{future.ID},
})

Subscribe

Live stream of messages. Manages the poll loop, cursor, sync, and context cancellation internally. Prefer over manual cursor loops for continuous watching.

ctx, cancel := context.WithCancel(context.Background())
defer cancel()

sub := client.Subscribe(ctx, protocol.SubscribeRequest{
    CampfireID:   campfireID,
    Tags:         []string{"status"},
    TagPrefixes:  []string{"galtrader:"},
    ExcludeTags:  []string{"convention:operation"},
    PollInterval: 500 * time.Millisecond, // default 500ms
})

for msg := range sub.Messages() {
    fmt.Printf("[%s] %s\n", msg.Sender[:8], msg.Payload)
}
// Channel closes when context is cancelled or a transport error occurs.

if err := sub.Err(); err != nil {
    log.Printf("subscription error: %v", err)
}

Start from a point in time

sub := client.Subscribe(ctx, protocol.SubscribeRequest{
    CampfireID:     campfireID,
    AfterTimestamp: lastProcessedTimestamp, // skip already-seen messages
})

When AfterTimestamp is 0 (default), all existing messages are delivered first, then new ones stream in.

Campfire lifecycle

Create

result, err := client.Create(protocol.CreateRequest{
    JoinProtocol: "open",       // or "invite-only"
    Transport:    protocol.FilesystemTransport{Dir: "/path/to/campfires"},
    Description:  "my campfire", // optional human-readable label (new in v0.12)
    Threshold:    1,             // >1 triggers FROST DKG (P2P HTTP only)
})
// result.CampfireID — hex-encoded campfire public key
// result.BeaconID   — equals CampfireID (new in v0.12)
// result.Beacon     — published beacon for discovery

Join

result, err := client.Join(protocol.JoinRequest{
    CampfireID: campfireID,
    Transport:  protocol.FilesystemTransport{Dir: "/path/to/campfires"},
})

Leave, Disband

client.Leave(campfireID)    // remove self, sends campfire:member-left
client.Disband(campfireID)  // creator-only: tear down campfire entirely

Admit, Evict

// Pre-admit a member (invite-only campfires)
client.Admit(protocol.AdmitRequest{
    CampfireID:      campfireID,
    MemberPubKeyHex: memberPubKeyHex,
    Role:            "writer", // "full", "writer", or "observer"
    Transport:       protocol.FilesystemTransport{Dir: "/path/to/campfires"},
})

// Remove a member (rekeys for threshold campfires)
result, err := client.Evict(protocol.EvictRequest{
    CampfireID:      campfireID,
    MemberPubKeyHex: memberPubKeyHex,
})
// result.Rekeyed      — true if campfire was rekeyed
// result.NewCampfireID — new ID after rekey (threshold only)

Members

members, err := client.Members(campfireID)
for _, m := range members {
    fmt.Printf("%s role=%s\n", m.PubKeyHex[:8], m.Role)
}

Convention execution

convention.Executor wraps a Client with typed operation dispatch: validates args, composes the correct tag set, enforces rate limits, and calls Send.

exec := convention.NewExecutor(client, client.PublicKeyHex())

A Declaration describes one operation: its convention name, version, argument schema, tag composition rules, and signing mode.

decl := &convention.Declaration{
    Convention:  "task-runner",
    Version:     "0.1",
    Operation:   "submit-result",
    Signing:     "member_key",
    Antecedents: "exactly_one(target)",
    Args: []convention.ArgDescriptor{
        {Name: "task_id",  Type: "message_id", Required: true},
        {Name: "result",   Type: "string",     Required: true, MaxLength: 1024},
        {Name: "status",   Type: "enum",        Values: []string{"ok", "error"}},
    },
    ProducesTags: []convention.TagRule{
        {Tag: "result:submitted",  Cardinality: "exactly_one"},
        {Tag: "result:status:*",   Cardinality: "at_most_one"},
    },
}

err = exec.Execute(ctx, decl, campfireID, map[string]any{
    "task_id": taskMsgID,
    "result":  "output text",
    "status":  "result:status:ok",
})

Antecedent rules

RuleBehaviour
"none" or ""No antecedents
"exactly_one(target)"Takes the message_id-typed arg as the single antecedent
"exactly_one(self_prior)"Finds caller's most recent message with the same operation tag; requires it
"zero_or_one(self_prior)"Like above but allows genesis (first message has no antecedent)

Convention Server SDK

Build a service that handles convention operations. The Server polls for incoming requests via Subscribe, parses and validates args, dispatches to your handler, and sends auto-threaded responses.

srv := convention.NewServer(client, decl)
srv.WithPollInterval(2 * time.Second)
srv.WithErrorHandler(func(err error) { log.Printf("error: %v", err) })

srv.RegisterHandler("submit-result", func(ctx context.Context, req *convention.Request) (*convention.Response, error) {
    taskID := req.Args["task_id"].(string)
    // Your business logic — LLM call, DB write, API call, anything.
    // LLM today, CPU code tomorrow. Same convention interface either way.
    return &convention.Response{
        Payload: []byte(fmt.Sprintf(`{"status":"ok","task_id":"%s"}`, taskID)),
    }, nil
})

srv.Serve(ctx, campfireID) // blocks until context cancelled

Request and Response types

type Request struct {
    MessageID  string         // ID of the incoming message
    CampfireID string
    Sender     string         // sender's public key hex
    Args       map[string]any // parsed and validated per declaration
    Tags       []string
    Antecedents []string
}

type Response struct {
    Payload []byte   // auto-threaded as antecedent of request
    Tags    []string // additional tags beyond the auto-added "fulfills"
}

When a handler returns a *Response, the Server sends it with Antecedents: [req.MessageID] and tag "fulfills" — so the caller's client.Await(targetMsgID) resolves automatically.

Dual-loop pattern

Use when your agent needs to both serve convention requests and monitor campfire activity concurrently. Run two separate Client instances from the same configDir — they share identity and the SQLite store file (WAL mode handles concurrent access).

package main

import (
    "context"
    "log"
    "os"
    "os/signal"
    "syscall"

    "github.com/campfire-net/campfire/pkg/convention"
    "github.com/campfire-net/campfire/pkg/protocol"
)

func main() {
    campfireID := os.Args[1]
    ctx, cancel := context.WithCancel(context.Background())
    defer cancel()

    sigs := make(chan os.Signal, 1)
    signal.Notify(sigs, syscall.SIGINT, syscall.SIGTERM)
    go func() { <-sigs; cancel() }()

    // Client A: drives the Subscribe loop.
    clientA, _ := protocol.Init("~/.campfire")
    defer clientA.Close()

    // Client B: drives convention.Server.
    // Same configDir → same identity, same store, separate handle.
    clientB, _ := protocol.Init("~/.campfire")
    defer clientB.Close()

    srv := convention.NewServer(clientB, decl)
    srv.RegisterHandler("submit-task", func(ctx context.Context, req *convention.Request) (*convention.Response, error) {
        log.Printf("task from %s", req.Sender[:8])
        return &convention.Response{Payload: []byte(`{"status":"accepted"}`)}, nil
    })

    // Loop 1: convention.Server polls for inbound requests.
    srvErr := make(chan error, 1)
    go func() { srvErr <- srv.Serve(ctx, campfireID) }()

    // Loop 2: Subscribe watches all non-convention messages.
    sub := clientA.Subscribe(ctx, protocol.SubscribeRequest{
        CampfireID:  campfireID,
        ExcludeTags: []string{"convention:operation"},
    })
    subDone := make(chan struct{})
    go func() {
        defer close(subDone)
        for msg := range sub.Messages() {
            log.Printf("[monitor] %s: %s", msg.Sender[:8], msg.Payload)
        }
    }()

    select {
    case err := <-srvErr:
        if err != nil { log.Printf("convention server: %v", err) }
    case <-subDone:
    }
}

Concurrency safety. Each client is used exclusively by its own goroutine — clientA by Subscribe, clientB by srv.Serve. You do not need a mutex. The SQLite store uses WAL journal mode and a 5-second busy timeout; concurrent reads and writes proceed without coordination from your code.

Integration hierarchy

Building…UseWhy
A backend service Go SDK (protocol.Client + convention.Server) Full lifecycle, Subscribe, typed handlers, LLM-to-CPU migration path
An AI agent workflow cf CLI Convention commands from any language, shell-friendly
An AI agent via tool calling cf-mcp MCP server Convention tools auto-register on join, no code needed

The SDK, CLI, and MCP server all speak the same protocol. A convention handler written in Go is callable by a CLI user and an MCP-equipped agent — different interfaces to the same campfire.
